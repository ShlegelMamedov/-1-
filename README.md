# вопросы 
1-Система контроля версий (VCS)
 Инструмент для фиксации изменений в файлах, позволяющий в любой момент вернуться к любой предыдущей версии. Используется в разработке ПО, написании документов, дизайне.
2-Репозиторий
 Хранилище проекта, содержащее все файлы и полную историю их изменений. Бывает локальным (на компьютере) и удалённым (на сервере, например, GitHub).
3-Коммит (commit)
 «Снимок» состояния файлов на определённый момент. Осмысленное сообщение коммита помогает быстро понять суть изменений — это критично для командной работы и отката ошибок.
4-Public vs Private репозиторий
 Public — виден всем: подходит для портфолио, открытых проектов, обучения.
 Private — доступен только вам и приглашённым: для личных или коммерческих проектов.
5-Ветка (branch) в Git
 Изолированная копия кода для разработки новой функции без риска сломать основную версию. В GitHub текущая ветка отображается в выпадающем списке над списком файлов (обычно main).
